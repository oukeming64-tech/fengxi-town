# 枫溪镇

给高金-上国会凯瑞商学院 22 期同学的一份毕业礼物。

[在线打开枫溪镇](https://oukeming64-tech.github.io/fengxi-town/)

![枫溪镇地图预览](gift-assets/map.png)

这里是一座可以打开的小镇。进去以后，可以看地图、看居民今天在哪里、看公告板、看今日小事，也可以等一天结束后读一份镇上小报。

## 怎么进入

最简单的方式是打开上面的在线链接，然后点击“打开枫溪镇”。

如果想在自己电脑上运行，下载这个仓库后直接打开 `index.html` 也可以；本地入口会进入 `town/index.html`。

## 模型文本

小镇不填 key 也能运行，会先按内置规则推进。想让居民短对话更丰富，可以打开“模型文本”，再进入“模型配置”填写自己的模型信息。

如果担心在线页面里填 key，建议在自己电脑上本地运行。key 留在自己的电脑和本地服务里，不需要写进仓库，也不会在页面上展示明文。

### 本地运行

先下载这个仓库。只看小镇和推进日常时，直接打开 `index.html` 即可；如果要让模型文本在本机工作，再在仓库目录启动本地模型服务：

```bash
node server.js
```

服务启动后，打开线上页面或本地的 `index.html` 都可以。页面默认会连接 `http://127.0.0.1:8787` 这个本地服务。

### key 怎么填

在小镇页面里：

1. 打开“模型文本”。
2. 点击“模型配置”。
3. 选择平台：`OpenRouter`、`OpenAI` 或 `兼容接口`。
4. 填“模型”，例如页面默认的 `deepseek/deepseek-v4-pro`，或你自己的模型名。
5. 填“接口地址”。OpenRouter 默认是 `https://openrouter.ai/api/v1/chat/completions`；OpenAI 默认是 `https://api.openai.com/v1/chat/completions`；兼容接口就填对方提供的 chat completions 地址。
6. 在 `API key 1` 里填自己的模型 key。如果有多个 key，可以继续填 `API key 2`、`API key 3`、`API key 4`。
7. 点击“保存配置”。页面会清空输入框，只显示识别到的 key 数量，不会回显明文。
8. 再推进一天，居民行动和短对话就会尝试走模型。

### 推荐：OpenRouter 四路并发

四路并发不是必须条件：只有一个 key 时，小镇仍能完整运行。如果希望 30 位居民的模型行动更快，并让每天同时产生不同方向的会话候选，可以准备 4 个 OpenRouter key。

1. 在 OpenRouter 的 key 管理页面创建 4 个不同的 key。不要把 key 发进聊天、截图或提交到 GitHub。
2. 在自己电脑上启动 `node server.js`，打开小镇的“模型配置”。
3. 平台选择 `OpenRouter`，模型可以保留默认的 `deepseek/deepseek-v4-pro`。
4. 没有设置环境 key 时，把 4 个 key 分别填进 `API key 1` 到 `API key 4`。
5. 如果本机环境里已经有 1 个 OpenRouter key，只需在页面填另外 3 个；本地服务会把环境 key 和页面 key 合并去重。
6. 点击“保存配置”。看到“页面 key x4”或“环境+页面 key x4”，就表示四路已经就绪。
7. 开启“模型文本”并推进一天。行动层会把 30 位居民分成 4 片并发处理；互动层会同时从互助回报、摩擦排挤、定调抢功和具体日常四个方向寻找候选，再由本地规则审核、去重，最多展示 3 段。

四路互动对全部 30 位居民一视同仁。服务会根据当前运行期间的近期出镜次数降低重复居民的优先级，同一天也尽量不重复同一个人；居民仍然按自己已经发生的行动和关系事实自然进入对话。

页面填写的 key 只保存在当前本地服务进程内存中；点击“清除页面 key”或关闭本地服务后就会消失。页面不会把它们写入仓库或居民日志。

也可以把 key 写在本机的 `.env.local`，然后再启动 `node server.js`。例如：

```bash
OPENROUTER_API_KEY=你的_key
OPENROUTER_MODEL=deepseek/deepseek-v4-pro
```

如果希望把 4 个 key 都放在本机环境文件里，可以使用逗号、分号或换行分隔：

```bash
OPENROUTER_API_KEYS=第一个_key,第二个_key,第三个_key,第四个_key
OPENROUTER_MODEL=deepseek/deepseek-v4-pro
```

如果用 OpenAI：

```bash
OPENAI_API_KEY=你的_key
OPENAI_MODEL=你的模型名
```

如果用兼容接口：

```bash
MORNING_TOWN_PROVIDER=compatible
COMPATIBLE_MODEL_URL=你的接口地址
COMPATIBLE_MODEL=你的模型名
COMPATIBLE_MODEL_API_KEY=你的_key
```

`.env.local` 只放在自己电脑上，不要上传到 GitHub。

## 枫溪镇里有什么

- 小镇地图：看看今天镇上有哪些地方正在发生事情。
- 镇上居民：看看居民现在在哪里、今天打算做什么。
- 桌面舞台：地图是主要观察界面，30 位居民会以不同小人形象在清晨、午后、夜里和归家四幕之间移动。
- 今日小事：一天推进之后，镇上会留下新的动静。
- 镇上对话：开启模型文本后，会出现居民之间的短对话。
- 公告板：可以看到设施、账务、招标和公开消息。
- 周报时间线：时间走远一点之后，可以回看阶段记录。
- 六十天阶段回顾：每完成 60 天会出现一次小镇阶段评价、关系亮点和已审核模型对话回顾，关闭后仍可从周报时间线重新打开。

## 当前状态

2026-07-14：当前版本为 v0.2.0。居民会根据已有性格参数使用不同的说话节奏和关注点，模型只能把这些差异用于表达已发生的事实，不能借此新增动机或事件。送出的小礼会来自当天行动或居民已有随身物品；同一对居民在一周内再次碰到旧分歧时会引用前次记录；账务场景改用“先顾哪件事、哪单快到期”等生活语言承接真实数值。

60 天真实模型运行暴露出的固定意象也已收敛：餐馆本地日志不再每天重复“油渍压住的纸条”，而会从黑板、菜筐、收音机、送货箱和桌边动作中轮换取证；内部作物分类会先转成居民可读名称。15 天复测后又收紧了例行价格、小黑板与路单、重复热饭三类主题的运行期上限，并让所有四路候选统一受限。模型候选台词只保留角色真正说出口的话；关系互动证据现在也能进入六十天对话归档。事实审核、四路并发、库存和关系数值边界不变。

节日地图包已补齐四套：春季换种日、春季清沟义工日、秋季丰收节和冬季落雪节。丰收节在秋季第 5–7 日显示暖色收成、共享长桌与摊位；落雪节在冬季第 4–5 日显示积雪、灯串与会堂聚餐氛围。四张地图都沿用 v0.1.9-a 的 3:2 画布、18 个热区、本地审核路线和图片加载失败回退，筹备与收尾阶段仍恢复默认地图。

节日居民参与也已接入本地行动策略：居民继续从已有的筹备、摆摊、送礼、致谢、共同劳动和晚宴活动里选择，节日只提供分时段加权。参与层级按节日轮换，保留一部分居民照常生活，不强制 30 人集中。地图和行为层都只读节日历与已审核活动事实，不让模型改节日日期、路线、热区、库存或关系数值。

节日结果会在每日结算后根据实际执行行动生成，并写入当天小报和本地周报。摘要记录参与人数、行动次数、主要结果、发生地点和证据引用；同一天重复处理不会产生重复记录，也不会再次写入库存、金币、关系或节日状态。没有开启模型时仍可完整运行。
